## Architectural Deviations From the Brief

The implementing operator confirmed the following deviations from the literal brief:

1. **`dev/active/<slug>/` is transient and `.gitignore`d**, not a parallel canonical surface. `docs/plans/<slug>.md` remains the durable plan; `MEMORY.md active_tasks` remains the cross-host pointer.
2. **Cadence is verification-gate-pass-driven**, not a fixed "every 3 micro-tasks". Binds to existing canonical events with crisp pass/fail semantics.
3. **Auto-activator is its own skill (`prompt-auto-activator`)**, not bolted onto `telemetry-guardian`. Preserves one-skill-one-purpose.
4. **`token-optimizer.targets: ["claude", "codex", "gemini"]`** as a literal list, not `["*"]`. Matches every other canonical skill in the factory.
5. **Hook record uses canonical event `user_prompt_submit`**, which is already in `_EVENT_ALIASES` for Claude and Codex (Gemini is `None`). No schema upgrade required.
6. **No `pre_turn` event added.** The brief's `pre_turn` was a synonym for what the schema already supports as `user_prompt_submit`.
7. **Sync deferred to operator.** Plan-mode constraint at the time of approval; respected after exit.
