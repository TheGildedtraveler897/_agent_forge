# Windows Manual Smoke Checklist (enterprise-readiness audit)

Date prepared: 2026-05-26
Branch under test: `audit/enterprise-readiness-2026-05-26` (or the RC re-cut from it)

Run this on a fresh **native** Windows VM (no WSL), PowerShell 5.1+. It covers the
full-parity additions from the 2026-05-26 audit. Use the newest bundle in
`exports/` (e.g. `agent-forge-suitcase-<stamp>.zip` + its
`...-deploy-and-bootstrap.ps1` sidecar).

## 1. One-shot deploy

1. Copy the suitcase `.zip` and its `-deploy-and-bootstrap.ps1` sidecar to the same
   short-path folder (e.g. `C:\af`).
2. In PowerShell:
   ```powershell
   powershell.exe -ExecutionPolicy Bypass -File .\agent-forge-suitcase-<stamp>-deploy-and-bootstrap.ps1 -BundleZip .\agent-forge-suitcase-<stamp>.zip -DestinationRoot C:\af
   ```
3. Confirm: ZIP unblocked, extracted, integrity check passed, and the resolver line
   prints a Python version (proves the **extracted** `_psutil.ps1` was dot-sourced —
   the sidecar ships standalone and validates Python post-extraction).
4. Confirm home flags forward: re-run with `-AllHosts -ClaudeHome C:\af\.claude`
   `-CodexHome C:\af\.codex` `-GeminiHome C:\af\.gemini` and verify no "unknown
   parameter" error (the `--gemini-home` gap fix).

## 2. Workstation bootstrap (install parity)

```powershell
cd C:\af\<bundle>\_agent_forge
powershell.exe -ExecutionPolicy Bypass -File .\scripts\bootstrap-workstation.ps1 -Install -Service 1
```
- **Python 3.10+ gate:** on a host with only Python < 3.10, confirm `Resolve-Python`
  rejects it with a clear message (not a silent pass). On a good host, confirm it
  prints the satisfied version.
- Confirm winget installs base tools and `npm install -g @anthropic-ai/claude-code`
  runs; `claude` resolves after reopening PowerShell.
- Confirm a log is written under `runtime\machine-setup\bootstrap-<stamp>.md`.

## 3. Project bootstrap (full parity)

```powershell
powershell.exe -ExecutionPolicy Bypass -File .\scripts\bootstrap-project.ps1 -Name demo -Define
```
- **Interactive CONOPS:** confirm the four `Read-Host` prompts appear and write
  `docs\CONOPS.md`. Re-run with `-NoDefine` and confirm no prompt.
- **Symlink fallback:** inspect `demo\.claude\CLAUDE.md`. Without Developer Mode /
  elevation it must be a **managed file copy** containing `Managed by Agent Forge`
  and pointing the import at the root adapter (not a crash). With Developer Mode /
  elevation it should be a real symlink.
- **projects.json:** confirm `demo` is registered (byte-identical shape to the `.sh`).
- `-Existing` (point at an existing dir) and `-WithLocalSkills` (creates
  `skills\projects\demo`) behave per help.

## 4. Verify + sync

```powershell
py -3 scripts\verify-agent-forge.py     # exit 0
```
- Confirm `omni_factory.py sync-claude --project demo` did NOT crash on the
  `.claude\skills\` symlink step (the WinError 1314 fix → copy fallback).

## 5. Onboarding render

- Open Claude Code at the project, run `/onboarding-guide`.
- **Acceptance gate:** Beat 0 (greeting + experience prompt) renders **inline in
  chat**, and the tour says **eight** beats.

## 6. Capture + record

- Save screenshots / command output under `runtime\validation\windows-2026-05-26\`.
- Set `windows.pass` in `runtime/validation-matrix.json` to `true` ONLY against
  captured evidence. Until then it stays `null` (not proven).
